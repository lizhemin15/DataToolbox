# 持久化存储功能测试指南

## 测试场景

### 测试1：首次启动 - 自动创建
**目的**：验证首次启动时自动创建持久化文件

**步骤**：
1. 确保 `apps/data-ontology/data-store.json` 不存在
2. 启动服务器
3. 观察控制台输出

**预期结果**：
```
持久化文件不存在，将创建新文件: .../apps/data-ontology/data-store.json
已创建默认管理员账号: admin/admin1234
数据已保存到: .../apps/data-ontology/data-store.json
数据本体池初始化完成 - 用户数: 1, 数据库配置数: 0
```

**验证**：
- `data-store.json` 文件已创建
- 文件包含默认管理员账号

---

### 测试2：添加数据库配置 - 自动保存
**目的**：验证添加数据库配置后自动保存

**步骤**：
1. 登录系统（admin/admin1234）
2. 添加一个新的数据库配置
3. 观察控制台输出
4. 检查 `data-store.json` 文件内容

**预期结果**：
```
数据已保存到: .../apps/data-ontology/data-store.json
```

**验证**：
- `data-store.json` 中包含新添加的数据库配置
- 数据库ID、类型、名称、连接信息都正确保存

---

### 测试3：重启服务器 - 自动加载
**目的**：验证重启后自动加载配置

**步骤**：
1. 确保已有数据库配置（参考测试2）
2. 停止服务器
3. 重启服务器
4. 观察控制台输出
5. 登录并检查数据库列表

**预期结果**：
```
已加载 1 个用户
已加载 1 个数据库配置
数据本体池初始化完成 - 用户数: 1, 数据库配置数: 1
```

**验证**：
- 登录后能看到之前添加的数据库配置
- 所有配置信息完整

---

### 测试4：编辑数据库配置 - 自动保存
**目的**：验证编辑配置后自动保存

**步骤**：
1. 选择一个数据库配置
2. 点击"编辑"按钮
3. 修改配置信息（如连接名称、主机地址等）
4. 保存修改
5. 检查 `data-store.json` 文件

**预期结果**：
```
数据已保存到: .../apps/data-ontology/data-store.json
```

**验证**：
- `data-store.json` 中的配置已更新
- 修改的内容正确保存

---

### 测试5：删除数据库配置 - 自动保存
**目的**：验证删除配置后自动保存

**步骤**：
1. 选择一个数据库配置
2. 点击"删除"按钮
3. 确认删除
4. 检查 `data-store.json` 文件

**预期结果**：
```
数据已保存到: .../apps/data-ontology/data-store.json
```

**验证**：
- `data-store.json` 中已不包含被删除的配置
- 其他配置不受影响

---

### 测试6：数据迁移 - 完整迁移
**目的**：验证完整文件夹迁移功能

**步骤**：
1. 停止服务器
2. 复制整个 `apps/data-ontology/` 文件夹到新位置
3. 在新位置启动服务器
4. 登录并检查数据

**预期结果**：
- 服务器正常启动
- 能够使用原有账号登录
- 所有数据库配置都存在

---

### 测试7：数据迁移 - 仅配置文件
**目的**：验证仅迁移配置文件的功能

**步骤**：
1. 停止服务器
2. 仅复制 `data-store.json` 到新环境
3. 在新环境启动服务器
4. 登录并检查数据

**预期结果**：
- 服务器正常启动
- 能够使用原有账号登录
- 所有数据库配置都存在

---

### 测试8：文件损坏 - 容错处理
**目的**：验证文件损坏时的处理

**步骤**：
1. 停止服务器
2. 手动修改 `data-store.json` 使其格式错误（如删除一个括号）
3. 启动服务器
4. 观察错误信息

**预期结果**：
```
加载持久化数据失败: 解析持久化数据失败: ...
已创建默认管理员账号: admin/admin1234
```

**验证**：
- 服务器能够启动（不会崩溃）
- 创建新的默认配置
- 记录错误日志

---

### 测试9：多数据库类型 - 混合配置
**目的**：验证支持多种数据库类型的混合配置

**步骤**：
1. 添加MySQL配置
2. 添加MongoDB配置
3. 添加Redis配置
4. 添加PostgreSQL配置
5. 重启服务器
6. 检查所有配置是否正确加载

**预期结果**：
- 所有类型的数据库配置都正确保存
- 重启后全部正确加载
- 不同类型的配置字段都正确

---

### 测试10：并发操作 - 数据一致性
**目的**：验证快速连续操作时的数据一致性

**步骤**：
1. 快速连续添加多个数据库配置
2. 快速修改和删除配置
3. 检查 `data-store.json` 文件
4. 重启服务器验证

**预期结果**：
- 所有操作都正确保存
- 文件内容完整且格式正确
- 重启后数据完整

---

## 自动化测试建议

可以编写脚本自动执行以上测试：

```bash
#!/bin/bash

# 测试1：首次启动
rm -f apps/data-ontology/data-store.json
./server &
sleep 3
kill $!

# 测试2-5：通过API测试
# 使用curl或其他HTTP客户端

# 测试6-7：文件操作测试
# 复制、移动文件并验证

# 测试8：容错测试
echo "invalid json" > apps/data-ontology/data-store.json
./server &
sleep 3
kill $!

# 生成测试报告
```

## 注意事项

1. 测试前备份重要数据
2. 在测试环境中进行，避免影响生产数据
3. 检查日志输出以获取详细信息
4. 验证文件权限设置正确
