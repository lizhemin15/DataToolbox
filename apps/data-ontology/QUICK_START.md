# 数据本体池 - 快速开始指南

## 🚀 5分钟快速上手

### 步骤 1: 启动服务器

```bash
# Windows
datatoolbox-server-windows-amd64.exe

# Linux
./datatoolbox-server-linux-amd64

# macOS
./datatoolbox-server-darwin-amd64
```

服务器启动后会显示访问地址，例如：
```
本地访问: http://localhost:8080
局域网访问: http://192.168.1.100:8080
```

### 步骤 2: 打开应用

在浏览器中访问：`http://localhost:8080`

然后找到并点击 **🗄️ 数据本体池** 应用。

### 步骤 3: 登录系统

使用默认管理员账号登录：
- **用户名**: `admin`
- **密码**: `admin1234`

### 步骤 4: 添加数据库

点击左侧 **+ 添加** 按钮，开始添加你的第一个数据库。

## 📋 常见数据库配置示例

## 🎯 实战场景

### 场景1: 快速查看生产数据库

**目标**: 查看生产MySQL数据库的用户表数据

1. 添加MySQL数据库配置
2. 点击"测试连接"确保能连接
3. 点击"保存"
4. 在左侧列表点击该数据库
5. 在右侧"数据表列表"中找到 `users` 表
6. 点击表名，即可看到前100条数据

**用时**: < 1分钟

### 场景2: 对比两个数据库的表结构

**目标**: 对比开发环境和生产环境的数据库

1. 添加开发数据库
2. 添加生产数据库
3. 分别点击两个数据库
4. 查看各自的表列表
5. 点击相同的表名查看数据

**用时**: < 2分钟

### 场景3: 验证数据迁移结果

**目标**: 确认从MySQL迁移到PostgreSQL的数据完整性

1. 添加源MySQL数据库
2. 添加目标PostgreSQL数据库
3. 对比表数量
4. 逐个查看重要表的数据
5. 确认数据已正确迁移

**用时**: < 5分钟

### 场景4: 多类型数据库统一管理

**目标**: 在一个界面管理不同类型的数据库

1. 添加MySQL（业务数据）
2. 添加MongoDB（日志数据）
3. 添加Redis（缓存状态）
4. 添加InfluxDB（监控数据）
5. 在左侧列表快速切换查看

**用时**: < 3分钟

## 💡 使用技巧

### 技巧1: 快速测试连接

在保存数据库配置前，先点击"测试连接"按钮验证配置是否正确。这可以避免保存错误配置。

### 技巧2: 使用只读账号

建议使用具有只读权限的数据库账号，避免误操作修改数据。

```sql
-- MySQL创建只读用户示例
CREATE USER 'readonly'@'%' IDENTIFIED BY 'password';
GRANT SELECT ON database_name.* TO 'readonly'@'%';
FLUSH PRIVILEGES;
```

### 技巧3: 命名规范

为数据库连接使用有意义的名称，例如：
- ✅ `生产-MySQL-用户库`
- ✅ `开发-PG-订单库`
- ✅ `测试-Mongo-日志库`
- ❌ `database1`
- ❌ `test`

### 技巧4: 分类管理

可以在连接名称中添加前缀来分类：
- `[生产]` - 生产环境数据库
- `[开发]` - 开发环境数据库
- `[测试]` - 测试环境数据库
- `[备份]` - 备份数据库

### 技巧5: 刷新数据

点击右上角的 **🔄 刷新** 按钮可以重新加载表列表和连接状态。

## ⚠️ 常见问题

### Q1: 连接失败怎么办？

**检查清单**:
1. ✅ 数据库服务是否正在运行？
2. ✅ 主机地址和端口是否正确？
3. ✅ 用户名和密码是否正确？
4. ✅ 数据库名是否存在？
5. ✅ 防火墙是否允许连接？
6. ✅ 数据库是否允许远程连接？

### Q2: 看不到表数据怎么办？

**可能原因**:
1. 表是空的（没有数据）
2. 用户没有查询权限
3. 表名拼写错误
4. 数据库连接已断开

**解决方法**: 点击"刷新"按钮重新加载

### Q3: 如何修改管理员密码？

目前版本暂不支持在界面修改密码。如需修改，需要：
1. 停止服务器
2. 修改 `server.go` 中的默认密码
3. 重新编译服务器
4. 重启服务器

### Q4: 数据库配置会保存吗？

⚠️ **重要**: 当前版本的数据库配置仅保存在内存中。重启服务器后，所有配置会丢失。

**建议**: 记录常用的数据库配置，或等待后续版本的持久化功能。

### Q5: 支持哪些数据库？

完整支持以下20+种数据库：
- **关系型**: MySQL, MariaDB, PostgreSQL, SQL Server, Oracle, 达梦, SQLite, DuckDB
- **分布式**: TiDB, CockroachDB
- **文档型**: MongoDB
- **KV存储**: Redis, Memcached
- **列式**: ClickHouse, Cassandra, HBase
- **时序**: InfluxDB, TimescaleDB
- **搜索**: Elasticsearch
- **图**: Neo4j

详见 [DATABASE_SUPPORT.md](DATABASE_SUPPORT.md)

生产环境必须修改默认管理员密码！

定期检查数据库访问日志。
