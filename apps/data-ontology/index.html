    可用工具
    
                    
      
                        
    • list_databases — 列出已配置的数据库（不含密码）
      • 
                        
    • get_tables — 获取指定数据库的表列表及连接状态
      • 
                        
    • describe_table — 获取表的列结构（字段名、类型、键信息等）
      • 
                        
    • execute_sql — 在指定数据库上执行任意 SQL（SELECT 返回数据，写操作返回影响行数）
      • 
                        
    • list_apis — 列出已配置的接口
      • 
                        
    • get_api_detail — 获取接口完整详情（含 SQL、参数定义、描述）
      • 
                        
    • call_api — 调用已配置的接口，传入接口 ID 和参数执行并返回数据

    🤖 AI助手
    
                        
                    
    
                    
    
                        
    
                            
    🤖
    
                            
    欢迎使用AI助手
    
                            
    使用 @数据库名 来引用数据库，AI将根据您的问题生成SQL查询并返回结果
    
                            
    
                                
    💡 智能上下文：
    
                                
    
                                    • 首次提问需要使用 @数据库名
    
                                    • 后续对话会记住数据库，无需重复 @
    
                                    • 使用新的 @ 会切换到新数据库
    
                                    • 点击右上角 ✕ 可清除上下文
                                
    
                                
    📝 查询示例：
    
                                
    
                                    • "@数据库 有哪些表？"
    
                                    • "users表有哪些字段？" ← 继续使用之前的数据库
    
                                    • "查看products表的前10条数据"
    
                                    • "查询价格大于100的产品"
                                
    
                                
    🔌 创建接口示例：
    
                                
    
                                    • "@数据库 帮我创建一个查询用户的接口"
    
                                    • "生成一个获取产品列表的API，按价格排序"
    
                                    • "创建接口：根据ID查询订单详情"
                                
    
                                
    🔧 数据治理任务示例：
    
                                
    
                                    • "@数据治理 @数据库 创建一个定时任务每天凌晨 2 点导入数据"
    
                                    • "生成一个交互任务：上传 Excel 后写入指定表"
    
                                    • 生成/修改后需在界面确认才会真正创建或更新任务

                            参数语法：
    
                            • #{参数名} - 预编译参数（推荐，用于数据查询，防止SQL注入）
    
                            • ${参数名} - 直接替换（用于表名、列名等）
    
                            ⚠️ DDL语句（CREATE/DROP/ALTER）必须使用 ${} 而不是 #{}
                            
                    
    
                
    
                
    
                    
    
                        
                        
                        请求将被原样转发到该URL，query参数和请求体会一并透传
