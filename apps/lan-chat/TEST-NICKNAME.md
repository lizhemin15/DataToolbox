# 昵称更新功能测试

## 修复的Bug

**问题描述：**
- 用户在一个标签页修改昵称后，昵称没有实时更新到聊天界面
- 其他用户也看不到昵称的变化

**修复内容：**
1. ✅ 服务器端：修改昵称后广播给所有用户
2. ✅ 前端：收到更新后刷新用户列表和聊天头部
3. ✅ 添加详细的日志输出便于调试

## 测试步骤

### 准备工作
1. 重新编译服务器（如果有Go环境）
   ```bash
   go build -o DataToolbox-Server.exe server.go
   ```

2. 启动服务器
   ```bash
   ./DataToolbox-Server.exe
   ```

3. 打开两个浏览器标签页
   - 标签页A: `http://localhost:8080/apps/lan-chat/`
   - 标签页B: `http://localhost:8080/apps/lan-chat/`

4. 两个标签页都按 F12 打开控制台查看日志

### 测试场景1：修改自己的昵称

**步骤：**
1. 在标签页A点击左上角的用户名
2. 输入新昵称，例如"张三"
3. 确定

**预期结果：**

**标签页A（自己）：**
- ✅ 左上角显示的昵称立即变为"张三"
- ✅ 头像显示"张"
- ✅ 控制台输出：
  ```
  昵称已更新: 用户xxxx -> 张三
  已通知服务器更新昵称
  ```

**标签页B（其他用户）：**
- ✅ 左侧联系人列表中，标签页A的昵称变为"张三"
- ✅ 如果正在与标签页A聊天，聊天头部显示"张三"
- ✅ 控制台输出：
  ```
  收到用户列表: [{id: "xxx", name: "张三"}, ...]
  用户列表已更新，当前在线用户: 张三, ...
  更新聊天头部，当前聊天对象: 张三
  ```

**服务器终端：**
```
客户端 xxx 更新昵称: 用户xxxx -> 张三
```

## 调试技巧

### 查看服务器日志
服务器会输出详细日志：
```
客户端 1708012345678901234 注册昵称: 用户5335
客户端 1708012345678901234 更新昵称: 用户5335 -> 张三
```

### 查看浏览器控制台
按 F12 打开控制台，会看到：
```
昵称已更新: 用户5335 -> 张三
已通知服务器更新昵称
收到用户列表: [{id: "xxx", name: "张三"}]
用户列表已更新，当前在线用户: 张三
更新聊天头部，当前聊天对象: 张三
```

### 如果昵称没有更新

**检查清单：**
1. ✅ 服务器是否已重新编译和启动？
2. ✅ 浏览器是否已刷新（Ctrl+F5 强制刷新）？
3. ✅ WebSocket 连接是否正常（状态显示为"在线"）？
4. ✅ 控制台是否有错误信息？

**调试步骤：**
1. 查看服务器日志，确认收到了 `update-name` 消息
2. 查看浏览器控制台，确认收到了 `peer-list` 消息
3. 检查 `peers` Map 中的数据是否正确
4. 在控制台输入 `peers` 查看当前用户列表

## 已知限制

1. **消息历史中的名字**：已发送的消息中显示的名字不会更新（这是正常的，显示发送时的昵称）
2. **离线用户**：只有在线用户会收到昵称更新
3. **断线重连**：断线重连后会获取最新的昵称
