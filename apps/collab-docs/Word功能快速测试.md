# Word功能快速测试指南

## 🚀 5分钟快速验证

### 步骤1：启动服务器（如果未运行）
```bash
cd z:\Documents\GitHub\DataToolbox
.\DataToolbox-Server.exe
```

### 步骤2：打开两个浏览器
- **浏览器A**（Chrome正常模式）
  - 访问：`http://localhost:8080/apps/collab-docs/`
  - 点击用户名，改为"张三"

- **浏览器B**（Chrome隐身模式或Firefox）
  - 访问：`http://localhost:8080/apps/collab-docs/`
  - 点击用户名，改为"李四"

### 步骤3：创建Word文档
**在浏览器A（张三）**：
1. 点击"新建文档"
2. 输入文档名："测试协作"
3. 选择"Word文档"

### 步骤4：李四加入
**在浏览器B（李四）**：
1. 应该看到文档列表中出现"测试协作"
2. 点击打开这个文档
3. 观察右上角：应该显示两个用户头像

### 步骤5：测试实时编辑
**张三（浏览器A）**：
1. 在编辑器中输入："你好世界"
2. 观察右上角是否显示两个头像

**李四（浏览器B）**：
1. ✅ **应该立即看到**："你好世界"

### 步骤6：测试双向编辑
**李四（浏览器B）**：
1. 在文档末尾输入："我是李四"

**张三（浏览器A）**：
1. ✅ **应该立即看到**："我是李四"
2. ✅ **应该看到**：李四的光标（不同颜色的竖线）

### 步骤7：测试格式化
**张三（浏览器A）**：
1. 选中"你好世界"
2. 点击工具栏的"B"（加粗）

**李四（浏览器B）**：
1. ✅ **应该看到**：文字变为加粗
2. ✅ **应该看到**：文字变为红色

## 🐛 如果测试失败

### 情况1：李四看不到张三的输入
**打开浏览器控制台（F12）**：

在张三的浏览器：
```
期望看到：
Word内容变化，广播更新: {ops: [...]}
```

在李四的浏览器：
```
期望看到：
应用Word远程更新: {ops: [...]}
Word更新成功
```

**如果没看到这些日志**：
1. 刷新两个浏览器
2. 重新打开文档
3. 检查WebSocket连接（应该显示绿色在线状态）

### 情况2：看不到光标
**检查**：
1. 两个用户是否在同一个文档？
2. 是否点击了编辑器（激活光标）？
3. 控制台是否有错误？

**解决**：
1. 确保两个浏览器都打开了同一个文档
2. 在编辑器中点击或输入文字
3. 查看控制台日志：`Word光标变化: {index: ...}`

### 情况3：光标位置不对
**这是正常的**：
- Quill的光标位置基于相对定位
- 滚动位置可能影响显示
- 5秒后光标会自动消失

### 情况4：格式化不同步
**可能原因**：
- 网络延迟
- 等待1-2秒

**解决**：
- 刷新李四的浏览器
- 应该能看到最终格式

## 🔧 高级调试

如果还有问题，执行以下命令：

### 检查WebSocket状态
```javascript
// 在控制台执行
console.log('WS状态:', ws?.readyState);
// 1 = 已连接, 0 = 正在连接, 2 = 正在关闭, 3 = 已关闭
```

### 检查当前文档
```javascript
console.log('当前文档:', currentDoc);
console.log('Quill:', quill);
console.log('在线用户:', onlineUsers);
```

### 检查光标
```javascript
console.log('Word光标:', wordCursors);
```

### 手动触发更新（测试用）
```javascript
// 模拟收到远程更新
applyRemoteUpdate({
    update: {
        type: 'word-delta',
        delta: { ops: [{ insert: '测试' }] }
    }
});
```
