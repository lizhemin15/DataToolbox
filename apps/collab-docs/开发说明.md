## 🏗️ 架构设计

### 整体架构
```
┌─────────────┐      WebSocket      ┌──────────────┐
│  浏览器 A    │◄──────────────────►│              │
│  (前端)      │                     │   Go服务器    │
└─────────────┘                     │  (后端)       │
                                    │              │
┌─────────────┐      WebSocket      │              │
│  浏览器 B    │◄──────────────────►│              │
│  (前端)      │                     └──────────────┘
└─────────────┘
```

### 通信流程
```
用户A编辑 → 前端捕获 → WebSocket发送 → 服务器
                                           ↓
用户B接收 ← 前端更新 ← WebSocket接收 ← 服务器广播
```

// 消息结构添加字段
type Message struct {
  // 原有字段...
  DocId     string      `json:"docId,omitempty"`

## 🔄 数据流详解

### 1. 创建文档流程
```
用户A点击新建
  ↓
前端生成文档对象
  ↓
广播doc-created消息
  ↓
服务器存储文档
  ↓
转发给所有用户（包括A）
  ↓
所有用户更新文档列表
```

### 2. 编辑同步流程
```
用户A编辑单元格
  ↓
input.oninput触发
  ↓
更新本地excelData
  ↓
广播doc-update消息
  ↓
服务器转发给其他用户
  ↓
用户B收到消息
  ↓
调用applyRemoteUpdate
  ↓
更新界面
```

### 3. 光标同步流程
```
用户A点击单元格
  ↓
input.onfocus触发
  ↓
广播doc-cursor消息
  ↓
服务器转发
  ↓
用户B收到
  ↓
调用updateCursor
  ↓
高亮显示单元格
```

function checkPermission(docId, action) {
  const doc = documents.find(d => d.id === docId);

  });
}

## 🚀 性能优化

### 1. 减少消息频率
```javascript
// 使用防抖避免频繁发送
let updateTimeout = null;
input.oninput = (e) => {
  clearTimeout(updateTimeout);
  updateTimeout = setTimeout(() => {
    broadcastUpdate(e.target.value);
  }, 300); // 300ms后发送
};
```

### 2. 增量更新
```javascript
// Word使用delta增量更新而非全量
quill.on('text-change', (delta) => {
  // 只发送变化部分
  broadcastMessage({
    type: 'doc-update',
    update: { type: 'word-delta', delta: delta }
  });
});
```
