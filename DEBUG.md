# 调试指南

## 局域网聊天调试

### 查看服务器日志

服务器现在会输出详细的日志信息，帮助排查问题：

```
客户端 1234567890 注册昵称: 用户3026
广播用户加入: 用户3026 (1234567890) 给 1 个其他用户
  -> 发送给: 9876543210
消息已发送: 1234567890 -> 9876543210
```

### 查看浏览器控制台

1. 按 F12 打开开发者工具
2. 切换到"控制台"标签
3. 查看日志输出：

```javascript
已注册，我的ID: 1234567890
收到用户列表: [{id: "9876543210", name: "用户5335"}]
新用户加入: {id: "1234567890", name: "用户3026"}
收到消息: {from: "9876543210", contentType: "text", content: "你好"}
```

### 测试步骤

1. **启动服务器**
   ```bash
   ./DataToolbox-Server.exe
   ```

2. **打开第一个浏览器标签页**
   - 访问: `http://localhost:8080/apps/lan-chat/`
   - 按F12打开控制台
   - 观察日志输出，应该看到：
     ```
     已注册，我的ID: xxxxx
     收到用户列表: []
     ```

3. **打开第二个浏览器标签页**
   - 访问: `http://localhost:8080/apps/lan-chat/`
   - 按F12打开控制台
   - 观察日志输出，应该看到：
     ```
     已注册，我的ID: yyyyy
     收到用户列表: [{id: "xxxxx", name: "用户xxxx"}]
     ```

4. **检查第一个标签页**
   - 应该自动显示新用户加入的消息
   - 控制台应该输出：
     ```
     新用户加入: {id: "yyyyy", name: "用户yyyy"}
     ```

5. **发送消息**
   - 在第二个标签页选择第一个用户
   - 发送消息"测试"
   - 第一个标签页应该收到消息
   - 两边控制台都应该有相应日志

### 常见问题排查

#### 问题1：看到自己在联系人列表中
**原因**: myId还未设置时就收到了peer-list
**解决**: 已添加代码在收到registered消息后清理自己

#### 问题2：新用户加入后第一个用户看不到
**原因**: 
- 服务器没有广播peer-join
- 或者客户端没有处理peer-join消息

**排查步骤**:
1. 查看服务器日志，确认有"广播用户加入"
2. 查看第一个标签页控制台，确认有"新用户加入"
3. 如果服务器没有输出，检查是否正确发送了register消息

#### 问题3：消息发送失败
**原因**:
- 目标客户端ID错误
- 目标客户端已断开连接

**排查步骤**:
1. 查看服务器日志的"消息已发送"或"目标客户端不存在"
2. 确认发送方和接收方的ID正确
3. 检查接收方控制台是否有"收到消息"日志

### 重新编译和测试

修改代码后需要重新编译：

```bash
# 停止旧服务器 (Ctrl+C)

# 重新编译
go build -o DataToolbox-Server.exe server.go

# 启动新服务器
./DataToolbox-Server.exe
```

然后**刷新所有浏览器标签页**（Ctrl+F5 或 Cmd+Shift+R）。

### 清理测试

如果遇到奇怪的问题，尝试完全清理：

1. 关闭所有聊天标签页
2. 停止服务器
3. 清除浏览器缓存（Ctrl+Shift+Delete）
4. 重启服务器
5. 重新打开标签页测试
