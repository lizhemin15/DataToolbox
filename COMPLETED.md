## 🛠️ 技术实现

### 前端技术
- 原生 JavaScript（无框架依赖）
- WebSocket 客户端
- FileReader API（文件处理）
- Drag & Drop API（拖拽功能）
- Clipboard API（粘贴功能）
- LocalStorage（数据持久化）

### 后端技术
- Go 1.21+
- gorilla/websocket v1.5.1
- Hub 模式管理客户端连接
- 消息路由转发
- 并发安全（RWMutex）

### 通信协议
```javascript
// 注册
{ type: 'register', name: '用户名' }

// 用户列表
{ type: 'peer-list', peers: [{id, name}, ...] }

// 用户加入
{ type: 'peer-join', peer: {id, name} }

// 用户离开
{ type: 'peer-leave', id: 'xxx' }

// 消息
{ 
  type: 'message',
  from: 'xxx',
  to: 'yyy',
  contentType: 'text|image|file',
  content: '...',
  timestamp: 1234567890
}
```

## 🎯 功能亮点

### 1. 拖拽发送（Drag & Drop）
**实现细节：**
- 监听 `dragover`、`drop`、`dragleave` 事件
- 拖拽时显示绿色高亮区域和提示文字
- 支持图片和文件自动识别
- 支持多文件批量拖拽

**用户体验：**
- 从文件管理器直接拖拽，无需点击按钮
- 视觉反馈清晰，操作直观
- 支持批量操作，效率更高

### 2. 粘贴发送（Clipboard）
**实现细节：**
- 监听 `paste` 事件
- 识别剪贴板中的图片数据
- 使用 FileReader 读取图片
- 显示预览模态框确认

**用户体验：**
- 截图后直接粘贴，无需保存文件
- 最快速的图片分享方式
- 工作流程最自然

### 3. 批量发送
**实现细节：**
- 检测拖拽的文件数量
- 显示文件列表预览
- 使用 setTimeout 间隔发送
- 避免阻塞和超时

**用户体验：**
- 一次性拖拽多个文件
- 清晰的文件列表预览
- 进度提示友好

### 4. Toast 提示系统
**实现细节：**
- 4种类型：info、success、warning、error
- 自动淡入淡出动画
- 3秒后自动消失
- 固定在顶部居中位置

**用户体验：**
- 操作反馈及时
- 不打断用户操作
- 视觉设计统一
